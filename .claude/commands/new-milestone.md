# /new-milestone

Use when the user gives sign-off to advance to a new milestone.

Steps:
1. Update `docs/milestones/MILESTONE_INDEX.md` — set previous milestone to Complete, set new milestone to Active
2. Read the new milestone file from `docs/milestones/`
3. Create the feature branch specified in the milestone file
4. Update `docs/working/plan.md` with the new milestone context
5. Report the milestone's objectives and first tasks to the user
