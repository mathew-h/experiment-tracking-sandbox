# /start-task

Use at the beginning of every work session.

Steps:
1. Read `docs/working/plan.md` — orient to current status
2. Read `docs/milestones/MILESTONE_INDEX.md` — confirm active milestone
3. Read the active milestone file from `docs/milestones/`
4. Run `git status` and `git log --oneline -10`
5. Confirm with the user what specific task they want to work on
6. Confirm scope before any implementation begins
7. Identify which sub-agent skill(s) are needed and read them
8. Update `docs/working/plan.md` with the task being started
